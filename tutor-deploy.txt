    -fill the url for clone repository

AUTOMATICLY DEPLOY
-from git version control, click manage to manage the repository that had been connnected
-go to pull or deploy section
-for able to clik update from remote to update repository in cpanel from github repository, make sure there is a valid .cpanel.yml in the repository. This file is for deployment configuration.

    MAKE .yml FILE 
    - make new file, name it ".cpanel.yml"
    - save it in root folder of that repository
    - the script:


---
deployment:
  tasks:
    - export DEPLOYPATH=/home/(web hosting user, ex:n1574345)/public_html/teknikinformatikaunri.xyz
    - /bin/cp index.html $DEPLOYPATH 
    - /bin/cp style.css $DEPLOYPATH
    - /bin/cp -R assets $DEPLOYPATH

Perintah /bin/cp -R assets $DEPLOYPATH mengcopy direktori "assets" beserta isinya ke $DEPLOYPATH.

Secara rinci, perintah tersebut memiliki arti sebagai berikut:

/bin/cp adalah perintah copy file/direktori di lingkungan Unix/Linux. Perintah ini akan menyalin file/direktori dari lokasi yang satu ke lokasi yang lain.
-R atau --recursive adalah opsi untuk menyalin direktori beserta isinya secara rekursif.
assets adalah direktori yang akan disalin.
$DEPLOYPATH adalah variabel lingkungan (environment variable) yang berisi path tujuan tempat direktori "assets" akan disalin.
Jadi secara keseluruhan, perintah /bin/cp -R assets $DEPLOYPATH akan menyalin direktori "assets" dan seluruh isinya ke lokasi yang ditentukan oleh variabel lingkungan $DEPLOYPATH.
